## `jq` が allowlist 未登録（真の missing-allow）

- **コマンド**: `[ -s "$BUF" ] && jq -c '...' "$BUF" || echo "(empty or missing)"` ほか、
  バッファ確認・JSON 整形で `jq` を多用。
- **原因**: `missing-allow`。`jq` が `permissions.allow` に無いためプロンプトが出る。`jq` は
  読み取り・整形用途で頻繁に使う無害コマンド。
- **推奨**: `"Bash(jq *)"` を allow に追加。
- **対象**: グローバル `~/.claude/settings.json`（＝ソース `ai-agents/settings/claude/settings.json`）。

## 検出器の誤検出（検出ロジックのバグ。要修正）

- **コマンド**: `git commit -F - <<'MSG' ... MSG` / `gh pr create --body "$(cat <<'BODY' ... BODY)"`
  / `git add \` の複数行フロー。
- **原因（誤検出）**: `permission-prompt-detect.sh` の `split_segments` が
  **改行で無条件分割**するため、
  1. ヒアドキュメント本文（コミットメッセージ・PR body の各行）が 1 行ずつ「コマンド」扱いされ
     すべて `missing-allow` に。
  2. `git add \`（行継続 `\`）が `\` 付きセグメントに割れる。
     実際にはこれらはプロンプトを生まない。`git *` / `gh *` は allow 済みで先頭セグメントは正しく通っている。
- **推奨（settings ではなく検出器の修正）**:
  - ヒアドキュメント（`<<'EOF' ... EOF` / `<<EOF`）本文を分割対象から除外する。
  - 行末 `\`（行継続）は分割せず前行に連結する。
  - `$(...)` / バッククォート内は 1 セグメントとして扱う（現状 best-effort 明記済みだが実害が出た）。
- **対象**: `ai-agents/settings/claude/hooks/permission-prompt-detect.sh`（コマンド書き換えではなく
  フック本体の改善）。
